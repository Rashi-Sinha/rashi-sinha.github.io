Foliage Generator Tool

Houdini FX 19.5

Procedural Content Generation

Houdini Digital Asset (HDA)

This Houdini Digital Asset enables artists to procedurally generate foliage, such as grass and bushes, for environment creation. It provides extensive controls for variation, including color palettes, grass and stem size, spread, and optional leaf generation. Users can also design custom bundle layouts (e.g., circular spreads or dense clusters) to create tailored foliage patterns. The final output is exported as low-poly cards with baked textures, ensuring optimal performance in game engines.

1. Curves

I began this project by following TECHARTWORLD’s course. The asset is built by defining vertical curves: points are scattered along a ground line and scaled using a ramp driven by their relative point numbers (VEX). Vertical lines are copied onto these points, and their lengths are controlled by this scale ramp. A bottom-to-top gradient attribute, computed from each point’s relative Y position, then drives noise in a Point VOP to add natural jitter to the curves.

Next, bending is introduced using another ramp based on each point’s relative X position. Two bend nodes define the angle range, and each curve is linearly interpolated between these angles to create smooth, controllable variation.

2. Adding Geometry

Once the base curves are built, geometry is added using a simple V-shaped profile and a Sweep node. The Sweep’s scale ramp controls thickness along each fiber, and additional variation comes from randomizing the pscale attribute per curve (based on prim number and clamped to a user-defined range).

For leafy variations, a basic leaf shape is created from a circle and processed through another Sweep to place leaves along the curves. Merging these leaves with the stems forms a fuller bush asset. A flatter V-shape generates wider leaves for an alternate variant. All options are routed through switch nodes so the user can easily toggle between styles. The final output is subdivided and has normals generated for clean shading.

3. Adding Color

The final stage focuses on adding color and texture. The base diffuse is created by blending two gradients (one along X and one along Y) combined with a palette of green tones. The Y-gradient reuses the same attribute that drove the curve noise, while the X-gradient is derived from each curve’s relative primitive number remapped to 0–1.

To add realism, randomized dark spots and subtle vertical fade lines are applied to each leaf or fiber. The spots are generated from random point positions, while the fade lines use the fractional part of the asset’s UVs (U coordinate) to distribute vertical streaks across each strand.

A low-poly plane is then placed beneath the foliage to capture and bake these colors into texture maps. Using the Maps Baker node, the final textures are exported for use with optimized low-poly cards.

4. Bundle Design

Once the asset design is complete, the next step is distributing it to populate an environment. A circle serves as a guide for arranging bundle components. Using remesh and a point wrangle, point distances are calculated and normalized to 0–1. This normalized attribute drives ramps for density and scale, determining how points are scattered and sized. The low-poly planes are then copied onto these points, inheriting their scale.

Normals for all cards in the bundle are generated from a source geometry created with a line. A Y-position ramp and revolve create a polygon around the bundle, normals are generated, and then transferred to the bundle primitives using an attribute transfer node.

The completed bundle can be exported and imported into a game engine. With proper shaders, materials, and high-resolution textures, it can convincingly represent grass or other foliage for environment population.

Note: Using the Apprentice version of Houdini imposes resolution limits and watermarks on exported textures, so the results shown are not fully sharp—but still demonstrate what the tool can achieve.
