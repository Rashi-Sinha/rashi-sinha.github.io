Soccer Ball : 3D Modeling, Texturing, Animation and Rendering

I started by getting familiar with the UI. It's a lot to get used to, but fortunately I had some practice with Maya and Blender, so it wasn't long before I was navigating through the software without pausing to remember the shortcuts. With this excercise, I was able to learn the basics of modeling and texturing in this software with a node-based network. I set up a simple keyframe animation for the ball with squishing and stretching. I also got a chance to explore Solaris to set up and assemble a scene for rendering including the USD that is created on the fly as the scene is set up. Followed this by adding and playing around with lights in the scene and finally rendering out the animation. The tutorial explained how to create a variation with rigid body simulation with the rbdsolver node instead of setting keyframes. Brickify : Houdini Digital Asset

This lesson was all about creating reusable procedural digital assets and definitely the most fun for me! It "brickifies" any 3D geometry that the user gives as input and builds it up like a lego creation. The "brick" is modeled separately. Then a 3D model is processed to get points from volume. Next, the "brick" is copied to those points with certain distance, color from the texture of the original 3D model is transferred to the bricks and a blast node is used to build up the brick version of the model from the ground up. The whole thing is converted to a Houdini Digital Asset with promoted parameters and input for custom geometry. Switches in the network were very handy for this part.

Procedural Terrain

Procedural content generation is fascinating to me. I had a lot of fun exploring the hieghtfield tools in Houdini. I started with a hieghtfield with some noise added to it. The next step was to separate out different parts of the terrain (usually by height) by creating masks for each tool, like for mountain peaks, plateau, valleys or river flows. I was able to add smaller details using the erode feature and marking the lower most valleys as water bodies. All that was left now was to add some trees. The scatter node was used to place random points on the plateau region and an inverted cone was copied to all those points as instances. The power of instancing was made clear to me with this. When used as a digital asset in Unreal, it allowed the user to replace these trees with differnt geometry and even a combimation of multiple diffrent geometries. The visualizer really helped bring things together to actually see each feature of the terrain with different colors within Houdini.

Procedural City : TOPS and PDG

Things got a little more complex with this one. I'm still working on getting a full grasp of how the TOPS network works and that's the fun part because I get to keep creating while exploring. On a higher level, the basics of this node network starts with tracing out a bock-grid using an image of a city map and separating each block as a primitive This is the input to the tops network, which can work on each block parallely to create what the rest of the network describes. A rop geometry node was used to model buildings and height variations were added based on the block (pdg) index. The local scheduler works on these blocks when it's cooked and we are able to look at each building depending on the block using a viewer in the main network. For the city to include all buildings at once, a waitforall node was used followed by a merge. City streets were created in a similar manner using the block grid as input and a separate viewer. The tutorial went over "wedge" nodes and how they can be used for creating variants using different maps. However, because I'm working with the Apprenctice version of Houdini and HDA doesn't work without Houdini engine, I wasn't able to test out the wedge node. I did end up finding a different map and plugging it in my main network to create a different city with the same network!
